技术架构方案

实现这款阅读器的 3 种技术路径

A

SwiftUI + WebKit（推荐）

SwiftUI 构建界面框架，WKWebView 渲染 EPUB/MOBI，PDFKit 渲染 PDF。现代架构，代码量适中，维护性好。

优势

  • UI 开发效率高，声明式语法
  • EPUB/MOBI 渲染质量好（基于 Web 引擎）
  • PDF 原生支持，性能优秀
  • macOS 12+ 即可运行

劣势

  • WebKit 渲染有一定内存开销
  • MOBI 格式需要预处理转换
B

AppKit + CoreRendering

纯 AppKit 界面，自建文本排版引擎（CoreText），PDFKit 渲染 PDF。完全原生，但开发量大。

优势

  • 完全原生，性能极致
  • 内存占用最低
  • 对 PDF 控制力最强

劣势

  • EPUB 渲染需从零实现
  • 开发周期长 3-5 倍
  • 维护成本高
C

Electron + Web 技术栈

用 HTML/CSS/JS 构建界面，Readium 处理 EPUB，PDF.js 渲染 PDF。跨平台潜力，但非原生。

优势

  • EPUB 支持最完善（Readium）
  • 跨平台潜力（可移植 Windows）
  • Web 生态丰富

劣势

  • 非原生体验，内存占用大
  • 启动慢，包体积大
  • 不符合 Mac 原生设计规范
