p class="subtitle">三种格式的解析和渲染方案

E

EPUB 处理

EPUB 本质是 ZIP 包（HTML + CSS + 图片），解析后注入自定义 CSS 控制排版，通过 WKWebView 渲染。

解析： unzip → 解析 content.opf 获取章节列表 → 逐章读取 XHTML
渲染：WKWebView 加载 HTML，注入字体/行距/主题 CSS
交互：JavaScript Bridge 处理选中文字、翻页、目录跳转
M

MOBI 处理

MOBI 格式复杂，建议导入时预转换为 EPUB 存储，后续直接读取转换后的文件。

导入时：调用命令行工具 calibre/ebook-convert 转为 EPUB
存储：转换后的 EPUB 保存到 App 沙盒，原文件保留但标记为"已转换"
优势：避免运行时解析 MOBI 的复杂性，复用 EPUB 渲染管线
P

PDF 处理

使用原生 PDFKit 框架，性能最优，支持文字选择、搜索、高亮。

渲染：PDFView 原生渲染，支持缩放、滚动
搜索：PDFDocument.findString() 原生全文搜索
高亮：PDFAnnotation 添加文字标注
限制：纯图片 PDF 无法选择文字（与原生 Preview 行为一致）
